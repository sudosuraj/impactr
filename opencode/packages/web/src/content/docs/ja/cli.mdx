Impactr CLI は、引数なしで実行すると、デフォルトで [TUI](/docs/tui) を開始します。

```bash
impactr
```

ただし、このページに記載されているようにコマンドも受け入れます。これにより、Impactr をプログラム的に操作できるようになります。

```bash
impactr run "Explain how closures work in JavaScript"
```

---

### tui

Impactr ターミナルユーザーインターフェイスを開始します。

```bash
impactr [project]
```

#### フラグ

| フラグ                                   | ショート                                | 説明                                                       |
| ---------------------------------------- | --------------------------------------- | ---------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`                                    | 最後のセッションを続行                                     |
| <nobr><code>{"--session"}</code></nobr>  |                                         | 続行時にセッションをフォーク (`-s` または `--fork` と併用) |
| <nobr><code>{"--continue"}</code></nobr> | <nobr><code>{"--session"}</code></nobr> | 続行するセッション ID                                      |
| <nobr><code>{"--prompt"}</code></nobr>   |                                         | 使用のプロンプト                                           |
| <nobr><code>{"--model"}</code></nobr>    | `-m`                                    | プロバイダー/モデルの形式で使用するモデル                  |
| <nobr><code>{"--agent"}</code></nobr>    |                                         | 使用するエージェント                                       |
| <nobr><code>{"--port"}</code></nobr>     |                                         | リッスンするポート                                         |
| <nobr><code>{"--hostname"}</code></nobr> |                                         | リッスンするホスト名                                       |

## コマンド

Impactr CLI には次のコマンドもあります。

---

### agent

Impactr のエージェントを管理します。

```bash
impactr agent [command]
```

---

### attach

`serve` または `web` コマンドを使用して起動された、すでに実行中の Impactr バックエンドサーバーにターミナルを接続します。

```bash
impactr attach [url]
```

これにより、リモート Impactr バックエンドで TUI を使用できるようになります。例えば：

```bash
# Start the backend server for web/mobile access
impactr web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
impactr attach http://10.20.30.40:4096
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                              |
| ---------------------------------------- | -------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |          | TUI を開始する作業ディレクトリ                                                    |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                            |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク（`--continue` または `--session` と使用）            |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `IMPACTR_SERVER_PASSWORD`）                   |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `IMPACTR_SERVER_USERNAME` または `impactr`） |

#### create

カスタム構成で新しいエージェントを作成します。

```bash
impactr agent create
```

このコマンドは、カスタムシステムプロンプトとツール構成を使用して新しいエージェントを作成する手順を示します。

利用可能なエージェントをすべてリストします。

```bash
impactr agent list
```

### auth

プロバイダーの資格情報とログインを管理するコマンド。

```bash
impactr auth [command]
```

---

#### login

Impactr は [Models.dev](https://models.dev) のプロバイダーリストを利用しているため、`impactr auth login` を使用して、使用したいプロバイダーの API キーを構成できます。これは `~/.local/share/impactr/auth.json` に保存されます。

```bash
impactr auth login
```

Impactr が起動すると、認証情報ファイルからプロバイダーがロードされます。また、環境またはプロジェクト内の `.env` ファイルで定義されたキーがあるかどうかも確認します。

---

#### list

認証情報ファイルに保存されているすべての認証されたプロバイダーをリストします。

```bash
impactr auth list
```

またはショートバージョン。

```bash
impactr auth ls
```

---

#### logout

資格情報ファイルからプロバイダーをクリアすることで、プロバイダーからログアウトします。

```bash
impactr auth logout
```

### github

リポジトリ自動化のための GitHub エージェントを管理します。

```bash
impactr github [command]
```

#### install

GitHub エージェントをリポジトリにインストールします。

```bash
impactr github install
```

これにより、必要な GitHub Actions ワークフローが設定され、構成プロセスがガイドされます。 [詳細はこちら](/docs/github)。

---

#### run

GitHub エージェントを実行します。これは通常、GitHub Actions で使用されます。

```bash
impactr github run
```

##### フラグ

| フラグ                                | 説明                                                |
| ------------------------------------- | --------------------------------------------------- |
| <nobr><code>{"--event"}</code></nobr> | エージェントを実行するための GitHub モック イベント |
| <nobr><code>{"--token"}</code></nobr> | GitHub 個人アクセストークン                         |

### mcp

モデルコンテキストプロトコルサーバーを管理します。

```bash
impactr mcp [command]
```

---

#### add

MCP サーバーを構成に追加します。

```bash
impactr mcp add
```

このコマンドは、ローカルまたはリモートの MCP サーバーを追加する手順を示します。

---

#### list

構成されているすべての MCP サーバーとその接続ステータスをリストします。

```bash
impactr mcp list
```

または、短いバージョンを使用してください。

```bash
impactr mcp ls
```

---

#### auth

OAuth 対応の MCP サーバーで認証します。

```bash
impactr mcp auth [name]
```

サーバー名を指定しない場合は、利用可能な OAuth 対応サーバーから選択するように求められます。

OAuth 対応サーバーとその認証ステータスを一覧表示することもできます。

```bash
impactr mcp auth list
```

または、短いバージョンを使用してください。

```bash
impactr mcp auth ls
```

---

#### logout

MCP サーバーの OAuth 資格情報を削除します。

```bash
impactr mcp logout [name]
```

---

#### debug

MCP サーバーの OAuth 接続の問題をデバッグします。

```bash
impactr mcp debug <name>
```

### models

構成されたプロバイダーから利用可能なすべてのモデルをリストします。

```bash
impactr models [provider]
```

このコマンドは、構成されたプロバイダー全体で利用可能なすべてのモデルを `provider/model` の形式で表示します。

これは、[設定](/docs/config/) で使用する正確なモデル名を把握するのに役立ちます。

オプションでプロバイダー ID を渡して、そのプロバイダーによってモデルをフィルターできます。

```bash
impactr models anthropic
```

#### フラグ

| フラグ                                  | 説明                                                            |
| --------------------------------------- | --------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | models.dev からモデルキャッシュを更新します。                   |
| <nobr><code>{"--verbose"}</code></nobr> | より詳細なモデル出力を使用します (コストなどのメタデータを含む) |

`--refresh` フラグを使用して、キャッシュされたモデルリストを更新します。これは、新しいモデルがプロバイダーに追加され、それを Impactr で確認したい場合に便利です。

```bash
impactr models --refresh
```

### run

プロンプトを直接渡して、非対話モードで impactr を実行します。

```bash
impactr run [message..]
```

これは、スクリプト作成、自動化、または完全な TUI を起動せずに迅速な回答が必要な場合に便利です。例えば。

```bash "impactr run"
impactr run Explain the use of context in Go
```

実行中の `impactr serve` インスタンスにアタッチして、実行ごとの MCP サーバーのコールドブート時間を回避することもできます。

```bash
# Start a headless server in one terminal
impactr serve

# In another terminal, run commands that attach to it
impactr run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### フラグ

| フラグ                                   | ショート | 説明                                                                                      |
| ---------------------------------------- | -------- | ----------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |          | 実行するコマンド。引数には message を使用します。                                         |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | 最後のセッションを続行                                                                    |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | 続行するセッション ID                                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |          | 続行時にセッションをフォーク (`--continue` または `--session` と併用)                     |
| <nobr><code>{"--share"}</code></nobr>    |          | セッションを共有する                                                                      |
| <nobr><code>{"--model"}</code></nobr>    | `-m`     | プロバイダー/モデルの形式で使用するモデル                                                 |
| <nobr><code>{"--agent"}</code></nobr>    |          | 使用するエージェント                                                                      |
| <nobr><code>{"--file"}</code></nobr>     | `-f`     | メッセージに添付するファイル                                                              |
| <nobr><code>{"--format"}</code></nobr>   |          | 形式: デフォルト (フォーマット済み) または json (生の JSON イベント)                      |
| <nobr><code>{"--title"}</code></nobr>    |          | セッションのタイトル (値が指定されていない場合は、切り詰められたプロンプトが使用されます) |
| <nobr><code>{"--attach"}</code></nobr>   |          | 実行中の impactr サーバー (http://localhost:4096 など) に接続します。                    |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Basic 認証パスワード（デフォルトは `IMPACTR_SERVER_PASSWORD`）                           |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Basic 認証ユーザー名（デフォルトは `IMPACTR_SERVER_USERNAME` または `impactr`）         |
| <nobr><code>{"--dir"}</code></nobr>      |          | 実行ディレクトリ、またはアタッチ時のリモートサーバー上のパス                              |
| <nobr><code>{"--variant"}</code></nobr>  |          | モデルバリアント（プロバイダー固有の推論レベル）                                          |
| <nobr><code>{"--thinking"}</code></nobr> |          | 思考ブロックを表示                                                                        |
| <nobr><code>{"--port"}</code></nobr>     |          | ローカルサーバーのポート (デフォルトはランダムポート)                                     |

### serve

API アクセスのためにヘッドレス Impactr サーバーを起動します。完全な HTTP インターフェイスについては、[サーバーのドキュメント](/docs/server) を確認してください。

```bash
impactr serve
```

これにより、TUI インターフェイスを使用せずに impactr 機能への API アクセスを提供する HTTP サーバーが起動します。 `IMPACTR_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `impactr`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

### stats

Impactr セッションのトークンの使用状況とコストの統計を表示します。

```bash
impactr stats
```

#### フラグ

| フラグ                                  | 説明                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------------------------------ |
| <nobr><code>{"--days"}</code></nobr>    | 過去 N 日間の統計を表示 (全期間)                                                           |
| <nobr><code>{"--tools"}</code></nobr>   | 表示するツールの数 (すべて)                                                                |
| <nobr><code>{"--models"}</code></nobr>  | モデルの使用状況の内訳を表示 (デフォルトでは非表示)。上位 N 件を表示するには数値を渡します |
| <nobr><code>{"--project"}</code></nobr> | プロジェクトでフィルタリング (全プロジェクト、空文字列: 現在のプロジェクト)                |

---

### export

セッションデータを JSON としてエクスポートします。

```bash
impactr export [sessionID]
```

セッション ID を指定しない場合は、利用可能なセッションから選択するように求められます。

---

### import

JSON ファイルまたは Impactr 共有 URL からセッションデータをインポートします。

```bash
impactr import <file>
```

ローカルファイルまたは Impactr 共有 URL からインポートできます。

```bash
impactr import session.json
impactr import https://opncd.ai/s/abc123
```

### web

Web インターフェイスを使用してヘッドレス Impactr サーバーを起動します。

```bash
impactr web
```

これにより、HTTP サーバーが起動し、Web ブラウザが開き、Web インターフェイスを通じて Impactr にアクセスします。 `IMPACTR_SERVER_PASSWORD` を設定して HTTP 基本認証を有効にします (ユーザー名のデフォルトは `impactr`)。

#### フラグ

| フラグ                                   | 説明                                    |
| ---------------------------------------- | --------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート                      |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名                    |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS 検出を有効にする                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS を許可する追加のブラウザーオリジン |

### acp

#### フラグ

| フラグ                                   | 説明                 |
| ---------------------------------------- | -------------------- |
| <nobr><code>{"--cwd"}</code></nobr>      | 作業ディレクトリ     |
| <nobr><code>{"--port"}</code></nobr>     | リッスンするポート   |
| <nobr><code>{"--hostname"}</code></nobr> | リッスンするホスト名 |

### uninstall

Impactr をアンインストールし、関連ファイルをすべて削除します。

```bash
impactr uninstall
```

#### フラグ

| フラグ                                      | ショート | 説明                                         |
| ------------------------------------------- | -------- | -------------------------------------------- |
| <nobr><code>{"--keep-config"}</code></nobr> | `-c`     | 構成ファイルを保持する                       |
| <nobr><code>{"--keep-data"}</code></nobr>   | `-d`     | セッションデータとスナップショットを保持する |
| <nobr><code>{"--dry-run"}</code></nobr>     |          | 削除せずに削除される内容を表示する           |
| <nobr><code>{"--force"}</code></nobr>       | `-f`     | 確認プロンプトをスキップする                 |

### upgrade

impactr を最新バージョンまたは特定のバージョンに更新します。

```bash
impactr upgrade [target]
```

最新バージョンにアップグレードするには。

```bash
impactr upgrade
```

特定のバージョンにアップグレードするには。

```bash
impactr upgrade v0.1.48
```

#### フラグ

| フラグ                                 | ショート | 説明                                                   |
| -------------------------------------- | -------- | ------------------------------------------------------ |
| <nobr><code>{"--method"}</code></nobr> | `-m`     | 使用されたインストール方法。curl, npm, pnpm, bun, brew |

## 環境変数

Impactr は環境変数を使用して構成できます。

| 変数                                  | タイプ   | 説明                                                                |
| ------------------------------------- | -------- | ------------------------------------------------------------------- |
| `IMPACTR_AUTO_SHARE`                 | ブール値 | セッションを自動的に共有する                                        |
| `IMPACTR_GIT_BASH_PATH`              | 文字列   | Windows 上で実行可能な Git Bash へのパス                            |
| `IMPACTR_CONFIG`                     | 文字列   | 構成ファイルへのパス                                                |
| `IMPACTR_CONFIG_DIR`                 | 文字列   | config ディレクトリへのパス                                         |
| `IMPACTR_CONFIG_CONTENT`             | 文字列   | インライン JSON 構成コンテンツ                                      |
| `IMPACTR_DISABLE_AUTOUPDATE`         | ブール値 | 自動更新チェックを無効にする                                        |
| `IMPACTR_DISABLE_PRUNE`              | ブール値 | 古いデータのプルーニングを無効にする                                |
| `IMPACTR_DISABLE_TERMINAL_TITLE`     | ブール値 | ターミナルタイトルの自動更新を無効にする                            |
| `IMPACTR_PERMISSION`                 | 文字列   | インライン化された json 権限設定                                    |
| `IMPACTR_DISABLE_DEFAULT_PLUGINS`    | ブール値 | デフォルトのプラグインを無効にする                                  |
| `IMPACTR_DISABLE_LSP_DOWNLOAD`       | ブール値 | LSP サーバーの自動ダウンロードを無効にする                          |
| `IMPACTR_ENABLE_EXPERIMENTAL_MODELS` | ブール値 | 実験モデルを有効にする                                              |
| `IMPACTR_DISABLE_AUTOCOMPACT`        | ブール値 | 自動コンテキスト圧縮を無効にする                                    |
| `IMPACTR_DISABLE_CLAUDE_CODE`        | ブール値 | `.claude` からの読み取りを無効にする (プロンプト + スキル)          |
| `IMPACTR_DISABLE_CLAUDE_CODE_PROMPT` | ブール値 | `~/.claude/CLAUDE.md` の読み取りを無効にする                        |
| `IMPACTR_DISABLE_CLAUDE_CODE_SKILLS` | ブール値 | `.claude/skills` のロードを無効にする                               |
| `IMPACTR_DISABLE_MODELS_FETCH`       | ブール値 | リモートソースからのモデルの取得を無効にする                        |
| `IMPACTR_FAKE_VCS`                   | 文字列   | テスト目的の偽の VCS プロバイダー                                   |
| `IMPACTR_CLIENT`                     | 文字列   | クライアント識別子 (デフォルトは `cli`)                             |
| `IMPACTR_ENABLE_EXA`                 | ブール値 | Exa Web 検索ツールを有効にする                                      |
| `IMPACTR_SERVER_PASSWORD`            | 文字列   | `serve`/`web` の基本認証を有効にする                                |
| `IMPACTR_SERVER_USERNAME`            | 文字列   | 基本認証ユーザー名 (デフォルト `impactr`) をオーバーライドします。 |
| `IMPACTR_MODELS_URL`                 | 文字列   | モデル設定を取得するためのカスタム URL                              |
