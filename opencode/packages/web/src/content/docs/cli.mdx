The Impactr CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
impactr
```

But it also accepts commands as documented on this page. This allows you to interact with Impactr programmatically.

```bash
impactr run "Explain how closures work in JavaScript"
```

---

### tui

Start the Impactr terminal user interface.

```bash
impactr [project]
```

#### Flags

| Flag                                        | Short | Description                                                             |
| ------------------------------------------- | ----- | ----------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr>    | `-c`  | Continue the last session                                               |
| <nobr><code>{"--session"}</code></nobr>     | `-s`  | Session ID to continue                                                  |
| <nobr><code>{"--fork"}</code></nobr>        |       | Fork the session when continuing (use with `--continue` or `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>      |       | Prompt to use                                                           |
| <nobr><code>{"--model"}</code></nobr>       | `-m`  | Model to use in the form of provider/model                              |
| <nobr><code>{"--agent"}</code></nobr>       |       | Agent to use                                                            |
| <nobr><code>{"--auto"}</code></nobr>        |       | Auto-approve permissions that are not explicitly denied                 |
| <nobr><code>{"--port"}</code></nobr>        |       | Port to listen on                                                       |
| <nobr><code>{"--hostname"}</code></nobr>    |       | Hostname to listen on                                                   |
| <nobr><code>{"--mdns"}</code></nobr>        |       | Enable mDNS discovery                                                   |
| <nobr><code>{"--mdns-domain"}</code></nobr> |       | Custom mDNS domain name                                                 |
| <nobr><code>{"--cors"}</code></nobr>        |       | Additional browser origin(s) to allow CORS                              |

## Commands

The Impactr CLI also has the following commands.

---

### agent

Manage agents for Impactr.

```bash
impactr agent [command]
```

---

#### create

Create a new agent with custom configuration.

```bash
impactr agent create
```

This command will guide you through creating a new agent with a custom system prompt and permission configuration. Anything you don't allow is denied in the generated agent's frontmatter.

#### Flags

| Flag                                        | Short | Description                                                                                                                                                                                                                |
| ------------------------------------------- | ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--path"}</code></nobr>        |       | Directory to write the agent file to (defaults to global or `.impactr/agent` based on the prompt)                                                                                                                         |
| <nobr><code>{"--description"}</code></nobr> |       | What the agent should do                                                                                                                                                                                                   |
| <nobr><code>{"--mode"}</code></nobr>        |       | Agent mode: `all`, `primary`, or `subagent`                                                                                                                                                                                |
| <nobr><code>{"--permissions"}</code></nobr> |       | Comma-separated list of permissions to allow (default: all). Available: `bash`, `read`, `edit`, `glob`, `grep`, `webfetch`, `task`, `todowrite`, `websearch`, `lsp`, `skill`. Anything omitted is denied. Alias: `--tools` |
| <nobr><code>{"--model"}</code></nobr>       | `-m`  | Model to use, in `provider/model` format                                                                                                                                                                                   |

Passing all of `--path`, `--description`, `--mode`, and `--permissions` runs the command non-interactively.

---

#### list

List all available agents.

```bash
impactr agent list
```

### attach

Attach a terminal to an already running Impactr backend server started via `serve` or `web` commands.

```bash
impactr attach [url]
```

This allows using the TUI with a remote Impactr backend. For example:

```bash
# Start the backend server for web/mobile access
impactr web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
impactr attach http://10.20.30.40:4096
```

#### Flags

| Flag                                     | Short | Description                                                                |
| ---------------------------------------- | ----- | -------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |       | Working directory to start TUI in                                          |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continue the last session                                                  |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | Session ID to continue                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |       | Fork the session when continuing (use with `--continue` or `--session`)    |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Basic auth password (defaults to `IMPACTR_SERVER_PASSWORD`)               |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Basic auth username (defaults to `IMPACTR_SERVER_USERNAME` or `impactr`) |

### auth

Command to manage credentials and login for providers.

```bash
impactr auth [command]
```

---

#### login

Impactr is powered by the provider list at [Models.dev](https://models.dev), so you can use `impactr auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/impactr/auth.json`.

```bash
impactr auth login
```

When Impactr starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

##### Flags

| Flag                                     | Short | Description                                          |
| ---------------------------------------- | ----- | ---------------------------------------------------- |
| <nobr><code>{"--provider"}</code></nobr> | `-p`  | Provider ID or name to log in to                     |
| <nobr><code>{"--method"}</code></nobr>   | `-m`  | Login method label to use, skipping method selection |

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
impactr auth list
```

Or the short version.

```bash
impactr auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
impactr auth logout
```

### mcp

Manage Model Context Protocol servers.

```bash
impactr mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
impactr mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
impactr mcp list
```

Or use the short version.

```bash
impactr mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
impactr mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
impactr mcp auth list
```

Or use the short version.

```bash
impactr mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
impactr mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
impactr mcp debug <name>
```

### models

List all available models from configured providers.

```bash
impactr models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
impactr models anthropic
```

#### Flags

| Flag                                    | Description                                                  |
| --------------------------------------- | ------------------------------------------------------------ |
| <nobr><code>{"--refresh"}</code></nobr> | Refresh the models cache from models.dev                     |
| <nobr><code>{"--verbose"}</code></nobr> | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Impactr.

```bash
impactr models --refresh
```

### run

Run impactr in non-interactive mode by passing a prompt directly.

```bash
impactr run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "impactr run"
impactr run Explain the use of context in Go
```

You can also attach to a running `impactr serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
impactr serve

# In another terminal, run commands that attach to it
impactr run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag                                     | Short | Description                                                                |
| ---------------------------------------- | ----- | -------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |       | The command to run, use message for args                                   |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continue the last session                                                  |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | Session ID to continue                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |       | Fork the session when continuing (use with `--continue` or `--session`)    |
| <nobr><code>{"--share"}</code></nobr>    |       | Share the session                                                          |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Model to use in the form of provider/model                                 |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agent to use                                                               |
| <nobr><code>{"--file"}</code></nobr>     | `-f`  | File(s) to attach to message                                               |
| <nobr><code>{"--format"}</code></nobr>   |       | Format: default (formatted) or json (raw JSON events)                      |
| <nobr><code>{"--title"}</code></nobr>    |       | Title for the session (uses truncated prompt if no value provided)         |
| <nobr><code>{"--attach"}</code></nobr>   |       | Attach to a running impactr server (e.g., http://localhost:4096)          |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Basic auth password (defaults to `IMPACTR_SERVER_PASSWORD`)               |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Basic auth username (defaults to `IMPACTR_SERVER_USERNAME` or `impactr`) |
| <nobr><code>{"--dir"}</code></nobr>      |       | Directory to run in, or path on the remote server when attaching           |
| <nobr><code>{"--port"}</code></nobr>     |       | Port for the local server (defaults to random port)                        |
| <nobr><code>{"--variant"}</code></nobr>  |       | Model variant (provider-specific reasoning effort)                         |
| <nobr><code>{"--thinking"}</code></nobr> |       | Show thinking blocks                                                       |
| <nobr><code>{"--auto"}</code></nobr>     |       | Auto-approve permissions that are not explicitly denied                    |

| <nobr><code>{"--hostname"}</code></nobr>    | Hostname to listen on                      |

| <nobr><code>{"--mdns-domain"}</code></nobr> | Custom mDNS domain name                    |

## Environment variables

Impactr can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `IMPACTR_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `IMPACTR_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `IMPACTR_CONFIG`                     | string  | Path to config file                               |
| `IMPACTR_TUI_CONFIG`                 | string  | Path to TUI config file                           |
| `IMPACTR_CONFIG_DIR`                 | string  | Path to config directory                          |
| `IMPACTR_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `IMPACTR_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `IMPACTR_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `IMPACTR_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `IMPACTR_PERMISSION`                 | string  | Inlined json permissions config                   |
| `IMPACTR_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `IMPACTR_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `IMPACTR_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `IMPACTR_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `IMPACTR_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `IMPACTR_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `IMPACTR_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `IMPACTR_DISABLE_MODELS_FETCH`       | boolean | Disable fetching models from remote sources       |
| `IMPACTR_DISABLE_MOUSE`              | boolean | Disable mouse capture in the TUI                  |
| `IMPACTR_FAKE_VCS`                   | string  | Fake VCS provider for testing purposes            |
| `IMPACTR_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `IMPACTR_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `IMPACTR_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `IMPACTR_SERVER_USERNAME`            | string  | Override basic auth username (default `impactr`) |
| `IMPACTR_MODELS_URL`                 | string  | Custom URL for fetching models configuration      |
